# 🤖 ArgoCD 운영 및 동기화 가이드 (GitOps)

이 문서는 **ArgoCD(GitOps)** 를 통해 AKS 쿠버네티스 클러스터에서 마이크로서비스를 배포하고 관리할 때 알아야 할 핵심 개념과 운영 지침을 정리합니다.

---

## 1. 🌟 GitOps 와 ArgoCD 의 핵심 개념

*   **동작 방식 (`Pull` 모델)**:
    *   GitHub Actions 가 빌드한 `.yaml` 명세표의 **상태를 감시**하여, **실제 클러스터(AKS)의 상태와 1:1로 일치**시킵니다.
    *   사용자가 직접 `kubectl apply` 를 칠 필요가 없으며, **오직 Git Push와 ArgoCD 화면 클릭** 만으로 배포를 완수합니다.

---

## 2. 🚦 ArgoCD 의 핵심 상태 (Status) 분석

ArgoCD 대시보드 화면에 뜨는 상태 아이콘의 의미는 다음과 같습니다.

| 상태 아이콘 | 상태명 | 의미 및 조치 방법 |
| :---: | :--- | :--- |
| **🟢 Healthy** | **정상 가동** | 파드들이 에러 없이 가동 중이며, 트래픽을 받을 준비가 되었습니다. |
| **🟡 Syncing** | **동기화 중** | 새로운 이미지나 설정이 주입되고 있으며, 컨테이너가 생성 중인 단계입니다. |
| **🟡 OutOfSync** | **변경 감지** | **깃허브의 코드**와 **서버의 현재 상태**가 다릅니다. (예: 최신 푸시 완료)<br>👉 **`[ SYNC ]`** 버튼을 누르면 서버에 덮어씌워집니다. |
| **🔴 Failed** | **가동 실패** | 설정 오류, 메모리 부족, DB 연결 실패 등으로 하트가 깨진 상태입니다. |

---

## 3. 🛠️ 핵심 운영 기능 (화면 조작법)

대시보드 상단에서 가장 많이 사용하실 핵심 버튼 3가지입니다.

### ① 🔄 **`[ SYNC ]` (동기화 - 배포 실행)**
*   **언제 사용하나?**: 깃허브에 최신 푸시를 한 뒤, 서버에 컨테이너를 실제로 띄우고 싶을 때 누릅니다.
*   *팁*: 앱 생성 시 `Automatic Sync` 를 꺼두셨다면 수동으로 이 버튼을 눌러야 명세표가 적용됩니다.

### ② ⚖️ **`[ DIFF ]` (차이점 비교)**
*   **언제 사용하나?**: `OutOfSync` 가 떴을 때, **예전 배포판과 이번 배포판의 코드가 정확히 어디가 바뀌었는지** (`+` / `-`) 대조해 보고 싶을 때 누릅니다.
*   배포하기 전 안정성을 최종 검토하는 용도로 매우 유용합니다.

### ③ ⏪ **`[ ROLLBACK ]` (이전 버전 복구)**
*   **언제 사용하나?**: 최신 배포한 버전에서 치명적인 에러가 나서, **즉시 5분 전의 정상 버전으로 되돌리고 싶을 때** 누릅니다.
*   ArgoCD 는 배포했던 커밋 히스토리 목록을 가지고 있으므로, 원하는 과거 시점을 선택해 `Rollback` 을 누르면 수 초 내에 원복됩니다.

## 4. 🚨 장애 대응 및 트러블슈팅 가이드

배포 후에 파드가 노란색/빨간색으로 멈춰있을 때 가장 흔히 겪는 현상입니다.

### ❓ **`ImagePullBackOff` (이미지가 안 떠요)**
*   **원인**: 도커 저장소(ACR)에 해당 버전 이미지(`ee669f...`)가 존재하지 않거나, 쿠버네티스가 ACR 에 접근할 권한이 없을 때 발생합니다.
*   **조치**: GitHub Actions 빌드가 ACR 에 정상적으로 푸시를 성공했는지 먼저 확인하세요.

### ❓ **파드 에러 로그(Log) 보는 법**
*   ArgoCD 대시보드에서 멈춰있는 파드(Pod) 아이콘을 클릭합니다.
*   상단 탭 중 **`[ Logs ]`** 탭을 클릭하시면, 스프링 부트나 파이썬 서버의 예외 에러 로그(`StackTrace`)를 화면에서 바로 읽을 수 있습니다!
